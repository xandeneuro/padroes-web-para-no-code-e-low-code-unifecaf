Fundamentos da Internet

Como a Web realmente funciona?

Antes de inspecionar qualquer site, precisamos entender o que acontece entre o momento em que você digita uma URL e o momento em que a página aparece na tela.

Internet ≠ Web

🌍

Internet

Infraestrutura física: cabos, roteadores, satélites e protocolos que conectam computadores ao redor do mundo. É a rede das redes, existe desde os anos 1960.

🕸️

Web (World Wide Web)

Um serviço que roda sobre a internet, criado por Tim Berners-Lee em 1989. É o conjunto de páginas e recursos acessíveis via navegador usando o protocolo HTTP.

Analogia: A internet é como o sistema de estradas do Brasil. A Web é como as lojas e casas que ficam às margens dessas estradas. Você pode usar as estradas para acessar outros serviços além das lojas — e-mail, streaming, jogos online — assim como a internet suporta outros serviços além da Web.

IP — O endereço real das máquinas

Todo dispositivo conectado à internet possui um endereço IP (Internet Protocol). É um número único — como o CEP de uma casa. Exemplo: 142.250.79.46 é um IP do Google. Você não precisa memorizar isso porque existe o DNS.

📡

IPv4

Formato: 4 grupos de números separados por ponto. Ex: 192.168.1.1. Suporta cerca de 4 bilhões de endereços — já estamos ficando sem.

🔮

IPv6

Formato novo, hexadecimal. Ex: 2001:0db8::1. Suporta uma quantidade astronômica de endereços — praticamente infinito.

DNS — A agenda telefônica da internet

O DNS (Domain Name System) traduz nomes legíveis como google.com para o IP numérico correspondente. Sem o DNS, você teria que memorizar IPs para acessar qualquer site.

Domínio, Hospedagem e URL

🏷️

Domínio

O nome único do seu site. Precisa ser registrado e renovado anualmente. No Brasil, o responsável é o NIC.br via registro.br.

🖥️

Hospedagem

O servidor onde seus arquivos ficam armazenados. Quando alguém acessa seu domínio, o servidor entrega os arquivos HTML, CSS, JS e imagens.

🔗

URL

O endereço completo de um recurso. Combina protocolo + domínio + caminho para localizar qualquer arquivo no servidor.

Anatomia de uma URL

https://www.unifecaf.edu.br:443/cursos/tecnologia
Protocolo (https) Subdomínio (www) Domínio (unifecaf) TLD / extensão (.edu.br) Porta (443 = HTTPS padrão) Caminho no servidor
HTTP vs HTTPS: o S significa Secure — a comunicação é criptografada. Sites sem HTTPS expõem dados do usuário. O navegador exibe um cadeado 🔒 para sites seguros. Todo site sério usa HTTPS hoje.

Modelo Cliente-Servidor

👤

Cliente (Browser)

Inicia as requisições, interpreta HTML/CSS/JS e renderiza a página visualmente. Exemplos: Chrome, Firefox, Safari, Edge.

🗄️

Servidor Web

Aguarda requisições, localiza os arquivos solicitados e os envia de volta como resposta. Exemplos: Apache, Nginx, Node.js.

O que o browser recebe e entende?

O servidor não envia uma "tela pronta" — ele envia arquivos de texto que o browser interpreta e converte em pixels.

📄

HTML

Conteúdo e estrutura. Define o que existe: textos, imagens, listas, vídeos, formulários. É o "esqueleto" da página.

Arquivo: index.html

🎨

CSS

Aparência e estilo. Define cores, fontes, espaçamentos, posicionamento e responsividade. É a "roupa" da página.

Arquivo: style.css

⚙️

JavaScript

Comportamento e interatividade. Animações, validações, chamadas de API. É o "sistema nervoso" da página.

Arquivo: app.js

Contexto da Disciplina

O que é Low-code & No-code?

Este é o tema central da disciplina. Antes de entrar no código, precisamos entender o panorama do desenvolvimento web atual e por que essas abordagens estão transformando o mercado.

Três formas de construir para a web hoje

👨‍💻

Full Code

Desenvolvimento tradicional. Você escreve cada linha manualmente em HTML, CSS, JavaScript e linguagens de back-end.

Máxima flexibilidade · Alta curva de aprendizado

🧩

Low-code

Ferramentas visuais que reduzem — mas não eliminam — a necessidade de código. Para casos específicos, ainda é possível escrever código customizado.

Equilíbrio entre agilidade e controle

🎛️

No-code

Ferramentas 100% visuais. Você cria aplicações completas sem escrever código, através de interfaces de arrastar e soltar.

Máxima velocidade · Possível limitação técnica

Low-code vs No-code — diferenças práticas

🎛️ No-code

Para quem? Empreendedores, designers, gestores.

  • Interface visual 100% drag & drop
  • Zero código necessário
  • Lançamento muito rápido
  • Templates prontos
  • Integrações via conectores visuais

Exemplos: Webflow, Wix, Bubble, Glide

🧩 Low-code

Para quem? Devs júnior, analistas, TI corporativa.

  • Interface visual + código quando necessário
  • Algum conhecimento técnico desejável
  • Alta produtividade
  • Maior controle sobre lógica de negócio
  • Integrações com APIs e bancos de dados

Exemplos: OutSystems, Mendix, Power Apps, Retool

Por que entender fundamentos web mesmo usando no-code?

🔧

Resolver limitações

Ferramentas no-code têm limites. Quem entende HTML/CSS consegue fazer ajustes via injeção de código — recurso presente no Webflow, Wix e outras plataformas.

🗣️

Comunicar com devs

Profissionais que usam no-code trabalham com equipes técnicas. Saber a linguagem comum — "requisição", "API", "DOM" — é fundamental para colaborar.

🚀

Fazer mais, mais rápido

A tendência do mercado é combinar as abordagens. Um bom profissional usa no-code para prototipar rápido e código para customizações avançadas.

🧠

Pensar como engenheiro

Entender como a web funciona por baixo muda a forma como você constrói. O resultado é um produto mais performático, acessível e bem estruturado.

🎯 Foco desta disciplina: Você vai aprender os fundamentos técnicos (HTML, CSS, JS) E dominar as principais ferramentas no-code do mercado — especialmente o Webflow. Ao final, saberá escolher a abordagem certa para cada projeto.

O mercado

O segmento de plataformas low-code e no-code cresce mais de 20% ao ano. Empresas como Google, Microsoft e Salesforce investem bilhões nesse setor. Analistas do mercado estimam que a maioria das novas aplicações empresariais será desenvolvida com esse tipo de ferramenta nos próximos anos.
🌊

Webflow

Vamos usar na disciplina. No-code para sites completos com CMS, hospedagem e animações avançadas.

💫

Bubble

No-code para aplicações web completas, com banco de dados, autenticação e lógica visual.

⚡

Power Apps

Low-code da Microsoft. Muito usado em ambientes corporativos integrados ao Office 365.

🌐 A Aba Network — onde tudo acontece

A aba Network registra cada arquivo que o browser baixou para montar a página. É como um log completo de todas as requisições HTTP.

Antes de começar: abra o DevTools, vá em Network e ative "Disable cache". Isso garante que você veja TODAS as requisições, sem nada sendo carregado do cache local.

O que tem dentro de cada requisição

📋

Headers

Cabeçalhos HTTP. Você verá o método (GET/POST), status code, IP do servidor, cookies. Procure: Content-Type, Cache-Control, Server

👁️

Preview / Response

O conteúdo retornado pelo servidor. Para APIs, o Preview mostra os dados em JSON formatado. Para HTML, mostra o código-fonte recebido.

⏱️

Timing

Quanto tempo cada fase levou: DNS Lookup, Connecting, Waiting (TTFB), Downloading. Ideal para identificar gargalos de performance.

🌍

Remote Address

O IP real do servidor que respondeu. Você pode pesquisar esse IP para saber em qual data center o site está hospedado (AWS, Google Cloud, Azure...).

📦 Cache — o arquivo que já veio antes

O cache é um mecanismo que guarda arquivos localmente para não precisar baixá-los novamente. Isso acelera sites — mas pode dificultar o desenvolvimento quando você faz mudanças que "não aparecem".

✅

Arquivo do cache

Na coluna "Status" você verá 304 Not Modified ou (disk cache). O arquivo veio do cache, não do servidor.

🔄

Forçar reload sem cache

Para baixar tudo do zero e ignorar o cache: Ctrl + Shift + R (Win/Linux) ou ⌘ + Shift + R (Mac).

💡 Na prática: quando você faz uma mudança no CSS e ela não aparece, muitas vezes é o cache. O Ctrl+Shift+R resolve na maioria dos casos.

Status Codes HTTP

✅ 2xx — Sucesso

200 OK — tudo certo
201 Created — recurso criado
204 No Content — sucesso sem dados

↩️ 3xx — Redirecionamento

301 Moved Permanently — URL mudou
304 Not Modified — use o cache
302 Found — redirect temporário

⚠️ 4xx — Erro do cliente

400 Bad Request — requisição inválida
401 Unauthorized — login necessário
403 Forbidden — sem permissão
404 Not Found — não existe

🔥 5xx — Erro do servidor

500 Internal Server Error — bug no servidor
502 Bad Gateway — problema no proxy
503 Unavailable — servidor fora

📟 Console — Comandos Simples

O Console é um terminal dentro do browser. Comandos básicos para usar hoje — sem funções complexas:

// Quantas imagens existem na página? document.images.length // Qual é o título da aba? document.title // Qual é a URL atual? document.URL // Quantos links existem? document.links.length // Listar os elementos h1 da página document.querySelectorAll('h1')
Atenção: nunca cole no Console código copiado de um lugar desconhecido. O browser executa tudo ali com acesso total à página aberta.

Mini-Hack ao Vivo

Modifique um Site Real com DevTools

Sem instalar nada. Tudo dentro do navegador. Ao recarregar a página, tudo volta ao normal — você não está alterando nada permanentemente.

Desafio 01 — Elements

🔤 Edite o título principal da página

Abra o Elements, encontre o <h1> mais evidente da página, dê duplo clique no texto e escreva qualquer coisa diferente. A mudança aparece instantaneamente.

⬤ Fácil
Desafio 02 — Styles

🎨 Mude uma cor pelo painel Styles

No painel Styles (à direita do Elements), clique no quadradinho colorido ao lado de qualquer background-color ou color. Um color picker abre — escolha uma nova cor. Pode ser o header, um botão, o fundo da página...

⬤ Fácil
Desafio 03 — Network

🔍 Encontre o arquivo mais pesado baixado

Na aba Network, clique na coluna "Size" duas vezes para ordenar do maior para o menor. Qual é o arquivo mais pesado? É uma imagem, vídeo ou script JS? Clique nele e veja a aba Preview para entender o conteúdo.

⬤ Fácil
Desafio 04 — Console

📊 Conte imagens e links com o Console

Abra o Console e execute os comandos abaixo, um de cada vez. Anote o que aparece.

document.images.length document.links.length document.title

Quantas imagens? Quantos links? O título bate com o que está na aba do browser?

⬤ Médio
Desafio 05 — Device Toolbar + Network

📱 Compare mobile vs desktop no Network

Ative o modo responsivo com Ctrl+Shift+M, selecione "iPhone 14 Pro" e recarregue. Compare a aba Network: o site baixou arquivos diferentes? Mudou a quantidade de requisições? Imagens menores foram carregadas?

⬤ Difícil
⚠️ Lembrete: Tudo que você muda no DevTools é local e temporário. F5 desfaz tudo. Você não está alterando nada no servidor.
